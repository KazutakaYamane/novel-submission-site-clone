# カクヨム機能の「削るリスト」(概念設計の前段・草案)

対象機能: 作品・エピソードの投稿・公開管理・閲覧(MVP コア機能)。
カクヨム本体が作品まわりに持つ概念を列挙し、クローンでの扱いを **採用 / 簡略化 / 削る** で判断する。

判断基準(`portfolio-project-knowledge.md` §1・§5.3 から):
1. バックエンドの設計判断(モデリング・状態遷移・不変条件)の見せ場になるか
2. 構築済みインフラ(ISR/SSR 使い分け・on-demand revalidation)を照らすか
3. 上記に寄与しない実装コスト(UI・バリデーション・表示分岐)を増やさないか

> **注**: カクヨム側の機能仕様は応募者(ヘビーユーザー)の観察で検証すること。

ステータス: **確定**(2026-07-03 迷いどころ5点を判断済み。末尾「確定の記録」参照)
最終更新: 2026-07-03

## A. 作品(Work)のメタデータ

| カクヨムの概念 | 判断 | 理由 |
|---|---|---|
| タイトル | **採用** | 必須。 |
| あらすじ(紹介文) | **採用** | 一覧・詳細ページの表示に必要。単純な text 属性でコスト最小。 |
| 公開状態(公開 / 非公開) | **採用** | 公開ライフサイクルの核。エピソード側の状態との**合成ルール(非公開作品内の公開エピソードは見えない)**が概念設計の主要な不変条件になる。 |
| 連載ステータス(連載中 / 完結済) | **採用** | 表示専用ではなく**ドメインルールを持つ**: 完結済の作品には新規エピソードを投稿できない、等の不変条件が状態モデルに参加する。 |
| ジャンル(固定選択・単一) | **簡略化して採用** | 固定マスタからの単一選択のみ。一覧のフィルタ軸として ISR ページ(ジャンル別一覧)の題材になる。カクヨムの全ジャンルは持たず 5〜6 個に絞る。 |
| タグ(自由入力) | **削る** | 多対多+正規化+タグページと、検索を削った本スコープでは活躍場面がない。タグモデリング自体は見せ場になり得るが、ジャンルで代替できる。意図的省略として ADR に記録。 |
| キャッチコピー | **採用** | カクヨム固有の特徴的な機能(一覧・詳細でタイトル横に出る短い惹句)であり、削るとクローンとしての再現度・カクヨム理解の印象を損なう。テキストのみの属性。**表示色は作品のテーマカラー(下記)に従う**。 |
| テーマカラー | **採用** | 作品単位の色設定(実物確認済み: 色を持つのはキャッチコピーではなく作品)。キャッチコピーがこの色の文字で表示される。プリセットからの選択とし、バリデーション対象にする。 |
| セルフレーティング(残酷描写等)/ R15 | **削る** | 表示フィルタリング(ゾーニング)要件が連鎖して重い。実サービスでは必須である旨を ADR に明記して省略。 |
| 限定公開(リンク限定) | **削る** | 公開/非公開の2値に絞る。推測不能 URL・アクセス制御の論点は面白いが、状態モデルが 2×3 に膨れる。 |
| 二次創作設定(公認二次創作) | **削る** | 原作マスタ・権利表示などカクヨム固有のエコシステム機能。クローンの主題外。 |
| 表紙・挿絵画像 | **削る** | UGC アップロード(S3 バケット・画像処理)が必要になる。削れば TODO フェーズ3 の UGC バケット自体を不要にできる。 |

## B. 作品の構造(章とエピソード)

| カクヨムの概念 | 判断 | 理由 |
|---|---|---|
| 章(チャプター)によるグルーピング | **採用** | Work > 章 > エピソードの2階層を再現する。**投稿者画面のエピソード・章位置の入れ替え機能を再現対象とする**ため、章は削れない。空章・章を持たない作品・章間移動といった境界条件が概念設計の議論対象になる。 |
| エピソードの表示順序・並び替え操作 | **採用** | 連載の本質+再現対象の中心機能。同一章内の並び替え、**章をまたいだエピソードの移動**、章自体の並び替えを含む。順序と所属をドメイン概念としてどう持つか(連番の振り直し戦略、移動の整合性)は設計の最大の見せ場の一つ。 |
| エピソードタイトル・本文 | **採用** | 必須。 |

## C. 執筆・公開管理(作者側)

| カクヨムの概念 | 判断 | 理由 |
|---|---|---|
| 下書き保存 | **採用** | 公開ライフサイクルの起点。コア。 |
| 公開 | **採用** | コア。**ISR on-demand revalidation のトリガー**であり、インフラ投資を照らす中心イベント。 |
| 予約公開(日時指定) | **採用** | 最大の設計見せ場(「状態か、公開日時が未来なだけか」「クエリで絞るか、バッチで倒すか」)。Redis キュー・ISR 再検証と絡む。 |
| 公開済みエピソードの改稿 | **採用** | 公開中コンテンツの更新→キャッシュ失効の導線。実際の作者が日常的に行う操作。 |
| 公開の取り下げ(公開→下書きに戻す) | **採用** | 状態遷移を一方通行にしない判断。読者が読んでいる最中の非公開化という境界条件が不変条件の議論を生む。 |
| エピソード削除(公開済み含む) | **採用** | 公開済みエピソードの削除も許す。実装は「直接削除」でも「まず下書きに戻してから削除、と投稿者をガイドする(=削除操作自体は下書きに限定)」でも可 — どちらを採るかは概念設計で決める。いずれにせよ**連載の歯抜け(第N話が消える)を順序モデルが許容する**必要がある点に注意。 |
| カクヨム記法(ルビ・傍点・改ページ) | **削る** | 記法パーサはスコープ外。本文はプレーンテキスト+改行のみ(エスケープ=XSS 対策は当然行う)。日本語組版の面白さより設計主題を優先。意図的省略として記録。 |
| 文字数カウント | **簡略化** | 保存時に算出する派生値として持つ程度(一覧・目次の「約N文字」表示用)。専用機能とはしない。 |

## D. 閲覧(読者側)

| カクヨムの概念 | 判断 | 理由 |
|---|---|---|
| 作品一覧(新着・更新順) | **採用** | コア。ソート主軸は「最終公開日時の降順」(ランキング不在の前提)。ISR ページ。 |
| 作品詳細=目次ページ | **採用** | コア。**章見出し付き**のエピソード一覧+各話公開日+連載ステータス。ISR ページ。 |
| エピソード本文ページ | **採用** | コア。ISR ページ。 |
| 前後エピソードへのナビ(prev/next) | **採用** | 順序概念の閲覧側の帰結。目次に戻らず読み続けられる連載サイトの必須動線。 |
| しおり・読書進捗(自動しおり) | **削る** | 読者ごとの状態管理が増える。読者側の書き込みの見せ場は「応援・お気に入り」(2機能目)が担う。 |
| 表示設定(フォントサイズ・テーマ) | **削る** | 純フロントの関心事で設計主題に寄与しない。 |
| PV カウント・アクセス解析(作者向け) | **削る** | 集計系はランキングと同じ理由で重い。 |

## E. 周辺機能(前回議論で確定済みの再掲)

| 機能 | 扱い |
|---|---|
| 応援・お気に入り(作品フォロー) | **2機能目として採用**(本リストの対象外。冪等性・カウンタ・キャッシュ無効化) |
| 応援コメント | **応援・お気に入り機能の拡張として採用**(簡略版: 投稿・一覧・作者削除のみ。返信・通知・通報は意図的省略)。実装順は応援・お気に入りの後(遅延時は切り捨て可能な位置)。エピソード本文は ISR、コメント欄は CSR 別フェッチに分離する(ISR キャッシュをコメント投稿で無効化しないため) |
| 認証 | 上記の前提として最小実装(メール+パスワード、Sanctum) |
| 検索 | 削る(ngram FULLTEXT の検討記録を ADR に) |
| レビュー(★+コメント) | 削る(評価軸が応援と重複) |
| ランキング・注目の作品 | 削る(集計バッチはスコープ超過) |
| フォロー通知・お知らせ | 削る(ファンアウト設計が重い) |
| 近況ノート | 削る |
| 自主企画・公式コンテスト | 削る |
| ギフト・収益化(ロイヤルティプログラム等) | 削る |

## 確定の記録(2026-07-03)

草案時点の「迷いどころ」5点に対する判断:

1. **章グルーピング → 採用**。投稿者側画面の「エピソードと章位置の入れ替え機能」を再現対象とするため。草案の「削る」を覆した。構造セクション(B)を全面改訂済み。
2. **ジャンル → 採用**(草案どおり)。タグは削る。
3. **公開取り下げ → 許可**(草案どおり)。
4. **公開済みエピソードの削除 → 許可**。実装形態は「削除操作は下書き限定とし、公開済みはまず取り下げるよう投稿者をガイドする」方式でも可。概念設計で確定する。
5. **連載ステータス → 採用**。表示属性ではなくドメインルール(完結済は新規エピソード投稿不可、等)を持たせる。

追加判断(2026-07-03):

6. **キャッチコピー → 採用**(草案の「削る」を変更)。カクヨム固有の特徴的な機能であり、削るとクローンとしての再現度が下がり採用担当者(カクヨム運営側のレビュアー)の心証を損なうため。「設計に寄与しないから削る」という草案の基準に対し、**カクヨムらしさの再現自体が応募上の価値を持つ**属性は例外とする。
   - **【2026-07-03 訂正】** 当初「キャッチコピーに文字色を設定する機能」と認識していたが、実物確認の結果、**作品にテーマカラーを設定し、キャッチコピーがその色の文字で表示される**仕様と判明。色は Work の属性(テーマカラー)として持ち、キャッチコピーはテキストのみとする。
7. **応援コメント → 応援・お気に入り機能の拡張として採用**(草案の「削る」を変更)。簡略版(投稿・一覧・作者削除のみ)に絞れば追加コストは CRUD 1セット分で軽い。返信・通知・通報は意図的省略として ADR に記録。ISR(本文)と CSR(コメント欄)の分離が ADR-FE の具体例になる副次効果もある。

### この確定が概念設計に持ち込む新しい論点

- **章の概念定義**: 章は必須か任意か(章を持たない作品を許すか)、空章を許すか、章に「タイトル」以外の属性はあるか。
- **順序モデルが2階層になる**: 章の並び順 × 章内のエピソード並び順。エピソードの章間移動時に両方の整合を保つ操作の設計。
- **削除と順序**: 公開済み削除により連載の歯抜けが正規の状態になる。「第N話」の番号を持つなら振り直しポリシーが要る(カクヨムは話数表示を持たない=順序のみ、を確認すること)。
- **完結済ステータスの遷移**: 完結済→連載中への差し戻しを許すか(許すのが自然。カクヨムも変更可能)。
