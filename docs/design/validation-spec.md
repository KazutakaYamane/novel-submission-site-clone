# 論理設計: バリデーション仕様

方針(Q9 確定): **実物(カクヨム)に寄せられるものは寄せ、確認できないものは常識的な運用値を決めて仕様として明記する。**
「出典」列がその区別。`運用値` はこの文書が正、`カクヨム準拠(要確認)` は実物で確認でき次第差し替える(差し替えは値の変更のみで設計に影響しない)。

ステータス: **確定(運用値ベース)**。テーマカラー54色の具体値(Q8)のみ実装時受領待ち。
最終更新: 2026-07-06

---

## 1. 共通ルール

- 文字数は **Unicode コードポイント数**(PHP `mb_strlen`)で数える。バイト数ではない。
- 入力は保存前に正規化する: 前後の空白・改行を trim(本文を除く)、改行コードは LF に統一(本文・あらすじ・コメント)。
- 本文・コメント等の自由テキストは**プレーンテキストとして保存**し、エスケープは表示側(Next.js のデフォルト挙動)で行う。API はサニタイズしない(データを壊さない)。
- 空文字と NULL: 任意項目の「未入力」は NULL で保存(空文字を保存しない)。
- バリデーション違反は 422 + Laravel 標準 errors バッグ(`api-contract.md` §5)。

## 3. 派生値の定義

| 値 | 定義 |
|---|---|
| episodes.character_count | **改行(LF)を除いた本文のコードポイント数**。保存(作成・編集・改稿)のたびに再計算。目次・一覧の「約N文字」表示に使う |
| 一覧の episode_count / cheer_count | published エピソードのみを対象に、ISR 再生成時に集計(DQ7) |

- バリデーション上限(body ≤ 50,000)は**改行を含む**生の長さ、表示用 character_count は**改行を除く**、と定義を分ける(前者はストレージ・転送の制約、後者は「読む量」の表現)。

## 4. ページネーション

| 一覧 | per_page | 出典 |
|---|---|---|
| 作品一覧(公開・ジャンル別) | 20 | 運用値 |
| コメント一覧 | 20 | 運用値 |
| お気に入り一覧 / 自分の作品一覧 | 20 | 運用値 |

`page` は 1 始まり。範囲外は空の data(404 にしない)。per_page はクライアント指定不可(固定)。
