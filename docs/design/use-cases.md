## 2. 検証結果

全ユースケースが現行モデル(domain-model + state-machine)の概念・関連・遷移・可視性ルールでたどれることを確認した。歩いて判明した不足は §3 としてモデル側に反映済み。

## 3. 検証で追加・明文化した規則

1. **「最終公開日時」は派生値**(U1): 一覧の更新順ソートキーは「配下 published Episode の published_at の最大値」。Work 自体の updated_at(メタ編集で動く)とは別物。メタだけ直しても一覧で浮上しない。
2. **prev/next は published のみをたどる**(U5): 読書順(章なし区画→章順)の上で draft / scheduled を飛ばす。`domain-model.md` §2 に反映済み。
3. **コメント欄・カウンタの扱い**(U11, Q7 確定): エピソードページのコメント欄・カウンタは CSR 別フェッチ(本文 ISR と分離)。**目次・一覧の応援数は ISR に焼き込み、時間ベースの定期 revalidate で遅れて反映**する(応援イベントで on-demand 失効はさせない)。
4. **個人化ページは SSR**(U8, U27): お気に入り一覧・作者管理画面はキャッシュ不可の SSR/CSR。ADR-FE の使い分けと整合。
